# Data Layers and Rules

This folder enforces a reproducible climate-data workflow.

## Structure
- `data/raw/`: Original downloaded files. Never manually edit.
- `data/staging/`: Normalized intermediate outputs.
- `data/processed/`: Final map-ready outputs generated by scripts.
- `data/sources/registry.csv`: Source inventory with license and notes.

## Rules
- Do not hand-edit `data/processed/*` values.
- Rebuild processed data using scripts in `scripts/`.
- Any new source must be added to `data/sources/registry.csv` first.
- Every production field must map to a documented source and method.
